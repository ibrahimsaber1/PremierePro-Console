# Contributing to Premiere Pro Console

First off, thank you for considering contributing to Premiere Pro Console! It's people like you that make this tool better for everyone.

## Code of Conduct

This project and everyone participating in it is governed by respect and professionalism. By participating, you are expected to uphold this standard.

## How Can I Contribute?

### Reporting Bugs

- **Ensure the bug was not already reported** by searching on GitHub under [Issues](https://github.com/ibrahimsaber1/PremierePro-Console/issues)
- If you're unable to find an open issue addressing the problem, [open a new one](https://github.com/ibrahimsaber1/PremierePro-Console/issues/new)
- Be sure to include a **title and clear description**, as much relevant information as possible, and a **code sample** or **executable test case** demonstrating the expected behavior

### Suggesting Enhancements

- Open a new issue with your suggestion
- Provide a clear description of the enhancement and why it would be useful
- Include examples of how the feature would work

### Pull Requests

1. Fork the repository
2. Create your feature branch (`git checkout -b feature/AmazingFeature`)
3. Commit your changes (`git commit -m 'Add some AmazingFeature'`)
4. Push to the branch (`git push origin feature/AmazingFeature`)
5. Open a Pull Request

**Please note:** I maintain this project alongside my full-time job, so while I'm committed to reviewing and merging contributions, response times may vary. I'll do my best to review PRs within 1-2 weeks, but please be patient during busy periods. Your contributions are highly valued!

## Development Guidelines

### Code Style

- Use clear, descriptive variable names
- Comment complex logic
- Follow existing code formatting patterns
- Test your changes with Adobe Premiere Pro before submitting

### Commit Messages

- Use the present tense ("Add feature" not "Added feature")
- Use the imperative mood ("Move cursor to..." not "Moves cursor to...")
- Limit the first line to 72 characters or less
- Reference issues and pull requests liberally after the first line
